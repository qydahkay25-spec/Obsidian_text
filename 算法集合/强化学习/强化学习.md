# 一、先理解强化学习的两个核心流派

强化学习算法几乎都属于两大类：

| 类型                     | 核心思想         | 代表算法          |
| ---------------------- | ------------ | ------------- |
| **Value-based（值函数方法）** | 学习每个动作有多好    | DQN           |
| **Policy-based（策略方法）** | 直接学习怎么做动作    | REINFORCE、PPO |
| **Actor-Critic（混合方法）** | 一个学策略，一个评估好坏 | DDPG、A3C、SAC  |
# 二、DQN 和 DDPG 的本质区别

|对比|DQN|DDPG|
|---|---|---|
|动作空间|**离散动作**|**连续动作**|
|方法类型|Value-based|Actor-Critic|
|输出|Q值|直接输出动作|
|网络数量|1个网络|2个网络|
# 三、为什么 DDPG 要两个网络？

因为它属于：

**Actor-Critic**

角色分工：

### Actor（演员）

负责：

决定动作

例如：

方向盘 = 15°

---

### Critic（评论员）

负责：

评价这个动作好不好

输出：

Q(s,a)

然后：

Critic指导Actor更新

---
